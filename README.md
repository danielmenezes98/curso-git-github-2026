\## \*\*3. Fluxo de trabalho Git local (resumo de comandos)\*\*



O instrutor apresentou um fluxo de trabalho para manter a organização:



1\. \*\*`git checkout -b NovaBranch`\*\* (Cria e entra na nova branch).

2\. \*\*Cria ou atualiza arquivos.\*\*

3\. \*\*`git add .`\*\* (Prepara os arquivos).

4\. \*\*`git status`\*\* (Verifica o que será enviado).

5\. \*\*`git commit -m "mensagem"`\*\* (Grava a alteração).

6\. git checkout main  (Volta para branch main)



\## 4. Fluxo de trabalho Github <> Local (projeto próprio ou da sua empresa)



1\. git clone (endereço do projeto)

2\. git checkout (nova branch)

3\. alterações de arquivos

4\. git status

5\. git add (arquivos)

6\. git status

7\. git commit -m (minha mensagem)

8\. git push origin (nova\_branch)

9\. abrir Pull request no Github para main

10\. excluir (nova\_branch) origin

11\. git checkout main

12\. git branch -D (nova\_branch)



\## 5. Fluxo de trabalho Github <> Local (projetos open-source)



1\. Fork do projeto para seu prórpio Github

2\. git clone (endereço do projeto fork)

3\. git checkout -b (nova\_branch)

4\. alterações de arquivos

5\. git status

6\. git add . (arquivos)

7\. git status

8\. git commit -m (mensagem)

9\. git push origin (nova\_branch)

10\. abrir Pull request no Github da branch fork para main do projeto original

11\. excluir (nova\_branch) origin

12\. git checkout main

13\. git branch -D (nova\_branch)
